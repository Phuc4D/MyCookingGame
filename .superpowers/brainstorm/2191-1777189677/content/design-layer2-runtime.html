Design — Layer 2: Runtime (MonoBehaviours)

Các GameObject sống trong scene. Chúng không chứa data — chỉ query RecipeDatabaseSO và thực thi logic.

FoodItem : MonoBehaviour

Gắn vào mỗi GameObject food trong scene. Chịu trách nhiệm duy nhất: track state hiện tại và cập nhật sprite.

Fields

FoodStateSO currentState
SpriteRenderer spriteRenderer

Behaviour

void ChangeState(FoodStateSO next)
 → cập nhật sprite tự động

StationBase : MonoBehaviour (abstract)

Base class cho mọi trạm làm việc. Quản lý slots và gọi RecipeDatabaseSO khi slot thay đổi.

RecipeDatabaseSO database   // inject qua Inspector
StationType stationType
List<FoodItem> slots         // food đang ở trong station

void AddFood(FoodItem food)
void RemoveFood(FoodItem food)
protected abstract void TryProcess()

Khi AddFood/RemoveFood → tự gọi TryProcess(). TryProcess() query database, nếu match thì xử lý.

InstantStation : StationBase

ChoppingBoard, PlatingTable

TryProcess():
 → query database
 → nếu match: destroy inputs,
   spawn output FoodItem
 → nếu không: không làm gì

TimedStation : StationBase

Stove, Oven

float progress
bool isCooking

TryProcess():
 → nếu match: start UniTask
   countdown duration giây
 → xong: spawn output

IngredientSource : MonoBehaviour

Gắn vào kệ nguyên liệu (ingredient shelf). Khi Chef tương tác → spawn FoodItem ở trạng thái Raw.

FoodStateSO ingredient  // set trong Inspector (fish.asset, meat.asset...)
void OnInteract() → Instantiate FoodItem với state = ingredient

Tại sao StationBase là abstract?

ChoppingBoard và Stove đều có slots, đều query database — nhưng khác nhau ở cách xử lý sau khi match: instant vs timed. Abstract class tránh duplicate code, mỗi subclass chỉ override đúng phần khác biệt.
