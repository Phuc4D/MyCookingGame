Design — Layer 3: Data Flow (Mọi thứ kết nối ra sao)

Một vòng hoàn chỉnh từ lúc Chef lấy nguyên liệu đến lúc ra món ăn.

Ví dụ: Nấu Fish Meal Tomato

Chef tương tác IngredientSource(fish)
→
Spawn FoodItem với state = fish.asset
Chef đặt fish vào ChoppingBoard
→
AddFood(fish) → TryProcess() → Database: [fish] + ChopBoard match? ✅ → destroy fish, spawn chopped_fish
Tương tự với tomato → chopped_tomato
→
Cùng flow, khác asset
Chef đặt 2 item vào Stove
→
AddFood(chopped_fish) → TryProcess: [chopped_fish] match? ❌
AddFood(chopped_tomato) → TryProcess: [chopped_fish, chopped_tomato] + Stove match? ✅ → start timer 3s → spawn cooked_fish_tomato
Chef đặt cooked item vào PlatingTable
→
TryProcess: [cooked_fish_tomato] + Plating match? ✅ → spawn fish_meal_tomato (Meal category)
Chef giao meal cho Serving Counter
→
ServingCounter query TryMatchRecipe(fish_meal_tomato) → tìm RecipeSO → cộng điểm, hoàn thành đơn

Multiplayer-ready như thế nào?

Station, FoodItem, RecipeDatabase hoàn toàn không phụ thuộc vào input hay số lượng player. Khi làm multiplayer sau, bạn chỉ cần thêm PlayerController thứ 2 gọi cùng các method AddFood(), RemoveFood() — phần còn lại không đổi gì cả.
