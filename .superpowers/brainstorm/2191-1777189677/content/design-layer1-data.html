Design — Layer 1: Data (ScriptableObjects)

Đây là nền tảng. Toàn bộ món ăn, công thức, biến đổi đều là SO assets — không hard-code gì trong code.

FoodStateSO

Mỗi "trạng thái" của food là 1 asset

string id
string displayName
Sprite sprite
FoodCategory category
  // Raw | Chopped | Cooked | Meal

Ví dụ assets: fish.asset, chopped_fish.asset, cooked_fish_tomato.asset, fish_meal_tomato.asset

TransformationSO

Mỗi "phép biến đổi" là 1 asset

FoodStateSO[] inputs
StationType requiredStation
float duration  // 0 = instant
FoodStateSO output

Ví dụ assets:
chop_fish: [fish] + ChopBoard → chopped_fish
cook_fish_tomato: [chopped_fish, chopped_tomato] + Stove(3s) → cooked_fish_tomato

RecipeSO

Dùng cho Order System — "đơn hàng"

string recipeName
FoodStateSO finalMeal
Sprite recipeCardSprite
int scoreValue
float timeLimit

Ví dụ: FishMealTomato.asset — yêu cầu fish_meal_tomato, 120đ, 60s

RecipeDatabaseSO

SO trung tâm — toàn bộ station query vào đây

List<TransformationSO> transformations
List<RecipeSO> recipes

// Methods:
bool TryGetTransformation(
  FoodStateSO[] inputs,
  StationType station,
  out TransformationSO result)

bool TryMatchRecipe(
  FoodStateSO meal,
  out RecipeSO result)

Tại sao FoodStateSO thay vì enum?

Enum đòi hỏi sửa code mỗi khi thêm món mới. SO asset = thêm món mới chỉ cần tạo file trong Unity Editor, không cần recompile. Designer có thể tự add món mà không cần lập trình viên.
